More than a canvas

👁️

Agent-native & accessible

Every interactive component projects a real, transparent DOM shadow node. A pure-canvas page is still read by screen readers and driven by Playwright / AI agents with getByRole().click() — something other canvas frameworks can't do.

🍃

Zero-DOM, zero reflow

The whole UI tree lives in one <canvas>. Layout, hit-testing and animation are pure math in JS/Workers — no browser reflow or style recalc. DOM-node count stays flat while animating (measured over CDP with compare:dom).

🔤

Typography that reflows

A cold/hot LayoutEngine: measure once, re-wrap on resize or browser zoom without re-measuring. Inline rich text, links, exclusion flow and streaming/typewriter incremental layout included.

⚡

Honest performance

Real benchmarks (headless Chrome): ~52fps for 10k on-screen entities, 60fps with viewport culling, 100k points through a single WebGL2 draw call. Every number reproduces with bun run benchmark.
